# 0x00 HAM 组合结构

## 组合基本用法

在 HAM 中，可以用 `{}` 来声明组合，例如：

```HAM
{ x = 1, y = 2 }
```

就代表一个组合，成为组合的声明式。可以通过 `.` 来获取组合中的元素

```HAM
{ x = 1, y = 2 }.x   // 1
```

组合的声明式可以作为“值”赋给键

```HAM
{
  comb = { x = 1, y = 2 },
  val = comb.x                // 1
}
```

从上面的例子我们还可以发现：

- 组合内的元素可以在声明后被直接使用，效果等同于声明时 `=` 后的表达式

- 组合内的元素不能在组合外访问

> 提示：上面的那些看起来像是变量的东西实际上不应该称为变量，而是一个表达式的别名，因为函数式下任何值都是不可变的。

HAM 的源代码文件本身就是一个巨大的组合，但是最外面的 `{}` 可以省略。

文件

```HAM
{
  a = 1,
  b = 2,
  c = 3
}
```

可以直接写成

```HAM
a = 1,
b = 2,
c = 3
```

请注意！组合不能简单地视作一个值，而是一些值的集合。在 HAM 里，一个键可以同时是值和组合

```HAM
x = ...,            // 后面会讲这里写了什么
a = x + 1,          // 把 x 作为值
b = x.element + 1   // 把 x 作为组合
```

你可以理解为在“赋值”的时候传递了一堆东西，但具体模式是怎么识别的不必深究，后面会提

## 键名的覆写

在组合中可以对同一个键名进行多次声明，称为覆写，在覆写后以后者为准

```HAM
x = 1,      // 1
x = x + 1   // 2
```

甚至可以“覆写组合内的元素”

```HAM
comb = { x = 1, y = 2 },
comb.x = 2
```

## 组合的提取和扩展

可以用类似覆写的方法扩展组合

```HAM
comb = { x = 1, y = 2 },
comb.z = 3
```

如果想要用一个组合扩展另一个组合，则可以使用提取元素的方法

提取元素（按键名把组合中的元素拿出来以直接使用）

```HAM
{x, y} = comb   // 相当于 x = comb.x, y = comb.y
```

用组合扩展组合

```HAM
comb1 = { a = 3, b = 4 },
comb.{a, b} = comb1
```

此外，组合的扩展也可以使用一种算符 `|>` ，上述组合扩展可以写为

```HAM
comb = comb1 |> comb
```

> 提示：用顺序匹配键的方式理解这种表达法，优先匹配 comb1 中的键

扩展时也可能会发生覆写

```HAM
comb2 = { a = 1, c = 2 },
comb1 = comb2 |> comb1      // { a = 1, b = 4, c = 2 }
```
