# 0x01 HAM 函数

函数是 HAM 的一等公民，HAM 用函数来表达运算。函数的声明式用 `() => ` 的形式表示，`()` 中为参数列表， `=>` 后为函数值

```HAM
a = () => 1,
f = (x) => x + 1,
add = (x, y) => x + y
```

当没有参数时，函数就是函数值，当只有一个参数时，参数列表的括号可以省略

```HAM
a = 1,
f = x => x + 1,
add = (x, y) => x + y
```

在函数后用 `()` 调用函数

```HAM
b = f(a),               // 2
c = add(a, b),          // 3
d = (x => x * x)(2)     // 4 函数的声明式可以直接调用
```

当函数的调用不合法时，函数值为空组合 `{}`

```HAM
a = add(1),     // {}
b = add(1,2,3)  // {}
```

> 提示：空组合其实就是无法使用的值，因为它没有元素，也不能当作值

是时候介绍 `|>` 算符的另一个用途了，除了扩展组合之外，`|>` 也可用于组合和值之间甚至连接两个值、两个既可以当作值也可以当作组合的东西（在 HAM 里这种东西广泛存在）

```HAM
x = 1 |> { x = 2 },   // 1 或 { x = 2 }
y = { x = 2 } |> 1    // 1 或 { x = 2 }
```

可见值和组合的顺序不重要

```HAM
x = 1 |> 2,   // 1
y = 2 |> 1    // 2
```

值和值的顺序是和覆写一样的

混合使用

```HAM
x = (1 |> { x = 1, y = 2 }) |> (2 |> { x = 2, z = 3 })
  // 1 或 { x = 1, y = 2, z = 3 }
```

可以将两个函数用 `|>` 连接，相当于新的函数，这个函数的行为是把参数分别带入左边和右边的函数，得到“值或者组合”，然后再把这些“值或者组合”按上述规则合并

写一个函数返回其参数个数对 3 取余的值（新语法会有介绍）

```HAM
argcMod3 <- (x, y, z) => argcMod3 |> (x, y) => 2 |> x => 1 |> 0,
a = argcMod3,                // 0
b = argcMod3(m),             // 1
c = argcMod3(m, n),          // 2
d = argcMod3(m, n, p),       // 0
e = argcMod3(m, n, p, q)     // 1
```

这里的 `<-` 表示允许递归的声明赋值，在 `<-` 后可以使用正在被定义的键

注意到 `e` 的声明中用了 4 个参数给 argcMod3 函数，为什么得到了 1？

这就要提到一种“不完全柯里化”了，意思就是当参数列表中的参数过多时，会从左往右取出需要数量的参数，剩下的留在括号里传给返回值（返回的函数），但是这不意味着任何函数都接受任意数量的参数，当括号里的参数不够时，函数会直接返回 `{}` ，而当函数返回非函数后括号里还有参数，也会返回 `{}`，这两种调用为非法调用。除了把参数写在一个括号里，连续使用括号进行调用也是可以的，有时这两种方法是等价的，有时则不同。

```HAM
// 等价形式推导过程
e = argcMod3(m, n, p)(q) |>
    argcMod3(m, n)(p, q) |>
    argcMod3(m)(n, p, q)
  = argcMod3(q) |>
    2(p, q) |>
    1(n, p, q)
  = 1 |>
    {} |>
    {}
  = 1
```

利用这些规则可以创建一些复杂的函数
