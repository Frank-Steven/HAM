# 0x04 HAM Δ 算符的分配律

Δ 算符 `<|` 对组合满足分配律（⊕ 表示任意二元算符）

```HAM
  (a <| b) ⊕ c
= a ⊕ c <| b ⊕ c
```

`c` 可以是任何表达式

```HAM
  (a <| b) ⊕ (c <| d)
= (a ⊕ (c <| d)) <| (b ⊕ (c <| d))
```

```HAM
  (a <| b) ⊕ x => x + 1
= (a ⊕ x => x + 1) <| (b ⊕ x => x + 1)
```

`<|` 对任何稳定的 Something 满足分配律

---

对于不稳定的 Something，则视为一个大函数。调用时，`<|` 两侧的表达式同时接收参数，每侧各自进行不完全柯里化，直到左侧稳定，用分配律变成稳定 Something。

---

## Something 标准式

```HAM
... [函数] <| [函数] <| [值] <| [组合] <| [函数] <| [函数] ...
```

- **稳定 Something**：标准形式中不含函数，仅剩 `[值] <| [组合]`
- **不稳定 Something**：标准形式中包含至少一个函数

`[值] <| [组合]` 和 `[组合] <| [值]` 是等价表达，具体规则遵循覆写原则。

---

## 调用时的不完全柯里化

调用不稳定 Something 时，`<|` 两侧各自独立进行不完全柯里化：

- 参数多于形参：取出所需数量，剩余参数传给返回值
- 参数等于形参：完全匹配，正常返回
- 参数少于形参：匹配失败，返回 `{}`
- 返回值非函数但仍有剩余参数：返回 `{}`

这个过程持续**直到左侧稳定**——即左侧不再有未消耗完参数的函数，所有左侧表达式已完全求值为值或组合。此时用分配律将所有结果合并为稳定 Something。

以 argcMod3 函数为例。

```HAM
argcMod3 <- 0 <| x => 1 <| (x, y) => 2 <| (x, y, z) => argcMod3
```

调用 `argcMod3(m, n, p, q)`（4 个参数），每个分支独立进行不完全柯里化：

```HAM
// 第一轮：4 个参数
0(m, n, p, q)                       = {}
(x => 1)(m, n, p, q)                = 1(n, p, q) = {}
((x, y) => 2)(m, n, p, q)           = 2(p, q) = {}
((x, y, z) => argcMod3)(m, n, p, q) = argcMod3(q) // 不稳定
// 即 {} <| {} <| {} <| argcMod3(q) = argcMod3(q)

// 第二轮：argcMod3(q)，1 个参数
0(q)                                = {}
(x => 1)(q)                         = 1 // 稳定
((x, y) => 2)(q)                    = {}
((x, y, z) => argcMod3)(q)          = {}
// 即 {} <| 1 <| {} <| {} = 1
```

---

两个函数用 `<|` 连接时，参数同时传给两个函数，直到左侧稳定，用分配律变成稳定 Something。

```HAM
  (a <| b) <| c
= (a <| c) <| (b <| c)
```

上述等式成立。
