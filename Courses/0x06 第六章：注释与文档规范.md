# 第六章 注释与文档规范

## 6.1 为什么需要注释
注释是写给人看的说明文字，不会被程序执行。
在编写代码的过程中，注释用来解释这段代码是什么、为什么这么写、有什么作用。

注释的主要作用：
- 让别人能快速看懂你的代码
- 让自己过一段时间后，还能理解当时的思路
- 标记暂时不用的代码，方便以后恢复
- 对复杂逻辑进行说明，降低理解成本
- 为工具生成说明文档提供依据

没有注释的代码，就像没有说明书的工具，很难使用和维护。

## 6.2 单行注释
HAM 使用 C++ / JavaScript 风格的单行注释，以双斜杠 `//` 开头。
从 `//` 开始到当前行末尾的所有内容均为注释，编译器会忽略。

单行注释适用于简短说明、行内备注。

## 6.3 块注释
HAM 使用 C++ / JavaScript 风格的块注释，以 `/*` 开头、`*/` 结尾。
块注释可以跨越多行，内部所有内容均为注释。

块注释不支持嵌套，主要用于一段说明、模块描述或暂时屏蔽一段代码。

## 6.4 文档注释
HAM 使用类 JSDoc 风格的文档注释，以 `/**` 开头、`*/` 结尾。
文档注释用于描述组合、键、接口的用途与约定，可被工具链提取生成 API 文档。

文档注释内部可以使用标记来描述结构，例如字段、参数、行为等，使组合结构与接口含义更加清晰。

## 6.5 注释最优实践
- 优先对复杂逻辑、组合结构添加注释，帮助阅读者快速理解
- 注释要说明意图，而不是重复代码已经表达的内容
- 对外提供的组合和接口，尽量使用文档注释，方便他人使用
- 保持注释简洁、准确，与代码逻辑保持一致
- 不要滥用注释，清晰的结构和命名本身就是最好的说明
- 可以使用块注释临时禁用一段代码，方便调试
